# 编辑器开发流程介绍

## 项目概述

编辑器(hmEditor)是基于CKEditor 4.0的二次开发项目，主要用于医疗文档编辑。项目保留了CKEditor的核心结构，并添加了业务特定逻辑和功能。

## 开发环境准备

1. **环境要求**
   - Node.js (推荐v12.x或以上)
   - Grunt CLI (全局安装: `npm install -g grunt-cli`)



3. **启动开发服务器**
   ```bash
   # 开发模式
   grunt def_test
   nodemon
   ```
   此命令会启动本地服务器，并在浏览器中打开示例页面(http://127.0.0.1:3071/demo/index.html)

## 目录结构

```
/hmEditor                 - 项目源代码目录
  /extensions             - 业务组件目录
    /base                 - 基础组件(主入口)
    /document             - 文档相关组件
    /docAi                - 文档AI相关组件
    ...
  /iframe                 - SDK主入口，用于创建iframe加载编辑器
  demo.html               - 本地测试页面
  README.md               - SDK使用文档
```

## 组件开发流程

### 1. 组件目录结构

每个业务组件必须遵循以下结构：

```
/extensions/{组件名}/
  /css/        - 组件样式文件（.less）
  /js/         - 组件JavaScript文件
  /tpl/        - 组件HTML模板
  module.js    - 组件入口文件
```

### 2. 创建新业务组件

1. 在`/hmEditor/extensions/`目录下创建新的组件目录
2. 按照规定的目录结构创建子目录和文件
3. 在`module.js`中定义组件并注册到命名空间

示例 module.js:
```javascript
// 注册组件
commonHM.component.register('myComponent', {});
```

### 3. 编写组件代码

#### JavaScript文件命名规则
- 文件名格式：`fn.*.js`，如`fn.method.js`、`fn.config.js`等
- 每个文件应当包含单一功能或相关功能集

#### 样式文件开发
- 使用LESS预处理语言
- 主样式文件命名为`ai.less`
- 组件专有样式应添加命名空间前缀，避免污染全局

#### HTML模板开发
- 模板文件存放在`tpl`目录下
- 使用artTemplate模板语法
- 模板将在构建时自动压缩并转换为JavaScript字符串

### 4. 组件开发最佳实践

1. **命名规范**
   - 组件名使用驼峰式，如`winDocModel`
   - 模板文件用语义化名称，避免数字编号

2. **代码组织**
   - 单一职责原则：一个函数只做一件事
   - 关注点分离：HTML/CSS/JS分离存放
   - 复用已有组件和功能，避免重复代码

3. **调试方法**
   - 使用浏览器开发者工具
   - 运行`grunt def_test`开启热重载功能
   - 修改文件后会自动重新构建并刷新浏览器

## 构建流程

### 开发构建
执行`grunt def_test`，流程如下：
1. 清理目标目录(`editorDist`)
2. 构建所有组件
   - 压缩HTML模板
   - 将模板转换为JS字符串
   - 合并模块JS文件
   - 编译LESS文件
3. 清理临时文件
4. 复制基础信息文件
5. 生成合并后的js和css文件
6. 启动开发服务器和监视文件变化

### 生产构建
执行`grunt release`，流程与开发构建类似，但不包含启动服务器和监视文件变化步骤。

## SDK使用方法

惠每电子病历编辑器提供了简单的SDK接口，可参考`hmEditor/README.md`文件了解详细用法。基本使用方式：

```javascript
// 创建编辑器（Promise方式 - 推荐）
HMEditorLoader.createEditorAsync({
    container: "#editorContainer",
    sdkHost: "服务器地址",
    editorConfig: {
        // 编辑器配置项
    }
})
.then(function(result) {
    // 编辑器初始化完成
    var editorInstance = result.instance;

    // 执行编辑器操作
    editorInstance.setContent("<p>编辑器内容</p>");
})
.catch(function(error) {
    console.error("编辑器初始化失败:", error);
});
```

## 常见问题与解决方法

1. **构建失败**
   - 检查Node.js版本是否兼容
   - 确保已全局安装grunt-cli
   - 检查是否有语法错误导致构建失败

2. **组件加载失败**
   - 检查组件目录结构是否正确
   - 确保module.js正确注册了组件
   - 查看浏览器控制台是否有错误信息

3. **样式问题**
   - LESS编译错误通常会在构建过程中报错
   - 使用浏览器开发者工具检查样式应用情况
   - 确保样式有正确的命名空间，避免与其他组件冲突
